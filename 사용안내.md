## ⚙️ 최초 설정 (한 번만)

### 1단계 — Supabase DB 테이블 생성
1. https://supabase.com/dashboard 접속 → 프로젝트 선택
2. 좌측 메뉴 **SQL Editor** 클릭 → **New query**
3. `supabase_schema.sql` 파일 내용을 전체 복사하여 붙여넣기
4. **Run** 버튼 클릭

테이블 5개가 생성됩니다:
- `domestic_quantities` (국내수량)
- `overseas_quantities` (해외수량·배송)
- `detail_ports` (세부포트)
- `overseas_holidays` (해외휴일)
- `network_changes` (네트워크 변경사항)

### 2단계 — 관리자 계정 비밀번호 설정
1. Supabase 대시보드 → 좌측 **Authentication** → **Users**
2. **Add user** → **Create new user**
3. Email: `sonojang@sinokor.co.kr`
4. Password: 원하는 비밀번호 입력 (8자 이상)
5. **Auto Confirm User** 체크 → **Create user**

> 또는: 관리자 페이지 접속 → "비밀번호 재설정 메일 받기" 버튼 사용

### 3단계 — 호스팅 (사내 직원 공유용)
가장 쉬운 방법: **Netlify Drop** (계정 불필요, 즉시 배포)

1. https://app.netlify.com/drop 접속
2. `calendar-survey-2027` 폴더를 통째로 드래그&드롭
3. 잠시 후 발급된 URL을 확인 (예: https://xxxx-yyyy.netlify.app)
4. 해당 URL을 카톡/메일로 사내 직원에게 공유

**Netlify 계정 가입시 장점** (선택):
- URL 변경 가능 (예: sinokor-calendar-2027.netlify.app)
- 파일 수정시 다시 드래그&드롭으로 업데이트

> 대체 호스팅: GitHub Pages, Vercel, Cloudflare Pages, Firebase Hosting 모두 가능

## 📋 사용 흐름

### 국내 팀장
1. 메인 URL 접속 → "국내 팀장" 카드 클릭
2. 본부 → 팀 선택 (2026년 실적이 참고로 표시됨)
3. 장금상선B / 흥아라인B 수량 입력
4. 입력자 이름 적고 제출

### 해외 주재원
1. 메인 URL 접속 → "해외 주재원" 카드 클릭
2. 상단에 본인 이름/이메일 입력 (4개 탭 공통)
3. 탭별로 입력:
   - **① 수량·배송정보**: 국가/지역 선택 → A/B 자동 표시 → 수량+배송정보 → 제출
   - **② 세부포트**: 일본처럼 여러 지역 분산 배송시 사용 (행 추가 가능)
   - **③ 다음년도 공휴일**: 국가 + 공휴일 여러 개 추가 후 일괄 제출
   - **④ 네트워크 변경**: 전년 대비 변경된 항목만 입력

### 관리자 (총무팀)
1. 메인 URL → "관리자" 카드 클릭
2. `sonojang@sinokor.co.kr` + 비밀번호로 로그인
3. 5개 탭으로 응답 조회
4. **📊 전체 엑셀 다운로드** 클릭 → 원본 양식 그대로 5개 시트 엑셀 파일 다운로드
5. **📄 현재 탭 CSV** — 활성 탭 데이터만 CSV로 다운로드

## 🛡 보안

- **publishable key**(`sb_publishable_...`)는 공개되어도 안전합니다. Row Level Security(RLS)로 보호됩니다.
- **익명 사용자**: INSERT만 가능 (응답 제출)
- **관리자(인증된 사용자)**: 전체 데이터 조회/수정/삭제 가능
- 관리자 페이지는 `sonojang@sinokor.co.kr` 이메일로만 로그인 가능

## 🔧 수정 방법

- **부서/팀 추가/변경**: `js/data.js`의 `DOMESTIC_ORG` 배열 수정
- **국가/지역 추가**: `js/data.js`의 `OVERSEAS_REGIONS` 배열 수정
- **2026년 실적값 변경**: `js/data.js`의 `prev_jangkum`, `prev_heunga` 수정
- **관리자 이메일 변경**: `js/config.js`의 `ADMIN_EMAIL`
- **다음년도 조사로 재사용**: `js/config.js`의 `SURVEY_YEAR` 변경 + `data.js`의 prev_* 값 업데이트

## ❓ 자주 묻는 질문

**Q. 같은 사람이 두 번 제출하면?**
A. 모든 응답이 저장되며, 엑셀 다운로드시 같은 본부/팀의 응답은 **최신 응답이 표시**됩니다. 관리자 페이지에서는 모든 응답이 시간 역순으로 보입니다.

**Q. 응답을 수정/삭제하고 싶다면?**
A. 본인이 다시 같은 본부/팀으로 제출하면 됩니다. 또는 관리자가 Supabase 대시보드에서 직접 삭제할 수 있습니다.

**Q. 응답이 모이는 위치는?**
A. Supabase 대시보드 → Table Editor 에서 실시간 확인 가능합니다.

**Q. 사이트 작동 안 함**
A. 브라우저 개발자 도구(F12) → Console 탭에서 에러 메시지 확인. 가장 흔한 원인은 (1) Supabase SQL 미실행, (2) RLS 정책 미설정.
