# 📄 과거 신청 수량 기능 — 적용 가이드

신청 폼(국내/해외)에 **📄 과거 신청 수량** 버튼이 추가되어, 클릭 시 작년 신청 수량을 표 형태로 확인할 수 있습니다. 매년 신청이 끝나면 자동으로 누적됩니다.

이 기능을 실제로 작동시키려면 아래 **3단계**만 진행하면 됩니다.

---

## ✅ Step 1 — DB 스키마 변경 (1분)

[Supabase SQL Editor](https://supabase.com/dashboard/project/wjfhybgwbtuqvylcaqfk/sql) 열기

1. 좌측 메뉴에서 **SQL Editor** 클릭
2. **+ New query** 클릭
3. 프로젝트 폴더의 **`supabase_v4_historical.sql`** 파일을 열어 전체 내용 복사 → SQL 에디터에 붙여넣기
4. **Run** 클릭

> 결과: `domestic_quantities` / `overseas_quantities` 테이블에 `survey_year` 컬럼 추가 + 집계 view 2개 생성

✅ "Success. No rows returned" 가 뜨면 정상

---

## ✅ Step 2 — 2026년 자료 일괄 적재 (1분)

같은 SQL Editor 에서:

1. **+ New query** 클릭
2. **`supabase_v4_data_2026.sql`** 파일 전체 내용 복사 → 붙여넣기
3. **Run** 클릭

> 결과: 2026년 국내 19행 + 해외 24행이 `survey_year = 2026` 으로 추가됨

✅ "Success. Rows: 43" 같은 메시지가 뜨면 정상 (재실행 시 중복 없이 갱신)

---

## ✅ Step 3 — Netlify 재배포 (1분)

변경된 파일:
- `domestic.html`, `overseas.html` (버튼 + 모달)
- `css/style.css` (모달 스타일)
- `js/domestic.js`, `js/overseas.js` (payload에 survey_year)
- `js/i18n.js` (다국어)
- `js/history-modal.js` (신규)

배포:
- **빠른 방법**: PowerShell에서 프로젝트 폴더로 가서 `netlify deploy --prod --dir .`
- **드래그**: https://app.netlify.com/sites/jolly-conkies-f8ee63/deploys 페이지에 폴더 통째로 드롭

## 🔄 내년 이후 운영

매년 신청이 끝나면 **추가 작업 없이 자동으로 누적**됩니다.

1. 한 해의 신청이 끝나면 → 그 해 데이터가 `survey_year = 2027` 로 DB에 누적되어 있음
2. 다음 해(2028) 조사 시작 시 → `js/config.js` 의 `SURVEY_YEAR` 를 `2028` 로 바꾸기만 하면 됨
3. 모달은 `survey_year < SURVEY_YEAR` 인 모든 연도를 자동으로 드롭다운에 표시
   → 2028 조사 중에는 모달에서 2026/2027 둘 다 선택 가능

## 🆘 문제 발생시

| 증상 | 원인 / 해결 |
|------|------------|
| 버튼 눌러도 모달이 안 뜸 | `js/history-modal.js` 가 배포에 포함되었는지 확인 |
| 모달 열리는데 "조회 오류" | Step 1 의 view 가 생성되지 않음. SQL Editor 에서 다시 실행 |
| 데이터가 안 보임 (빈 화면) | Step 2 의 INSERT 가 안 됐음. `select count(*) from domestic_quantities where survey_year=2026` 로 확인 |
| 합계가 이상함 | view 가 같은 (country, region) 의 회사별 행을 합산함. 의도된 동작 |
