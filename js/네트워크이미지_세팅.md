# 네트워크 이미지 업로드 기능 — 최초 1회 세팅

> 이번에 admin 페이지에서 직접 장금/흥아 네트워크 이미지를 업로드할 수 있게 만들었어.
> Supabase Storage에 저장되므로 **버킷 1개 + 정책 2개**만 한 번 만들면 끝.
> 매년 조사 전엔 admin 페이지에서 새 이미지 업로드만 하면 됨 (Netlify 재배포 불필요).

---

## ✅ Step 1 — Storage 버킷 생성 (30초)

1. [Supabase Storage](https://supabase.com/dashboard/project/wjfhybgwbtuqvylcaqfk/storage/buckets) 열기
2. 우측 상단 **`New bucket`** 클릭
3. 입력:
   - **Name**: `network-images`
   - **Public bucket**: ✅ **체크** (해외 주재원이 익명으로 볼 수 있어야 하므로)
4. **`Create bucket`** 클릭

---

## ✅ Step 2 — 정책 추가 (2분)

Public bucket이라도 storage.objects는 RLS가 적용됨.
**anon에 SELECT** 정책(list/download)을 명시해야 해외 주재원이 이미지를 볼 수 있고,
**authenticated에 INSERT/UPDATE/DELETE** 정책으로 관리자가 업로드 가능.

1. 좌측 메뉴 **Storage** → 방금 만든 `network-images` 버킷 클릭
2. 상단 **`Policies`** 탭 클릭
3. **`New policy`** → **`For full customization`** 선택 (세 번 반복)

### 정책 1: 익명 SELECT (해외 주재원 페이지에서 조회용) ⚠ 누락 시 토글해도 같은 이미지 나옴

- **Policy name**: `anon select network-images`
- **Allowed operation**: ✅ SELECT
- **Target roles**: `anon`
- **USING expression**:
  ```
  bucket_id = 'network-images'
  ```
- **Save**

### 정책 2: 관리자 INSERT

- **Policy name**: `auth insert network-images`
- **Allowed operation**: ✅ INSERT
- **Target roles**: `authenticated`
- **WITH CHECK expression**:
  ```
  bucket_id = 'network-images'
  ```
- **Save**

### 정책 3: 관리자 UPDATE + DELETE

- **Policy name**: `auth modify network-images`
- **Allowed operation**: ✅ UPDATE, ✅ DELETE (둘 다 체크)
- **Target roles**: `authenticated`
- **USING expression**:
  ```
  bucket_id = 'network-images'
  ```
- **WITH CHECK expression** (UPDATE에 필요):
  ```
  bucket_id = 'network-images'
  ```
- **Save**

## 🆘 문제 발생시

| 증상 | 원인 / 해결 |
|------|------|
| admin에서 "업로드 실패: new row violates row-level security" | INSERT/UPDATE 정책의 **Target roles가 `authenticated`** 이어야 함 (anon ❌) |
| 업로드는 성공했는데 해외 페이지에 안 보임 / 흥아 눌러도 장금 이미지가 나옴 | **Step 2 정책 1 (anon SELECT)** 추가됐는지 확인. 누락 시 list가 빈 결과를 줘서 둘 다 폴백 이미지로 나옴 |
| 위 정책도 다 추가했는데 안 보임 | 버킷이 **Public**으로 설정됐는지 확인. Storage → 버킷 → Configuration → "Public bucket" 토글 |
| 업로드 후 미리보기가 옛날 이미지 | 브라우저 캐시. F5(강력 새로고침: Ctrl+Shift+R) — 코드에 캐시 무력화 쿼리(`?t=업로드시각`)가 이미 들어가 있어서 정상 케이스에선 자동 갱신됨 |
| "Bucket not found" | Step 1의 버킷 이름이 정확히 `network-images` 인지 확인 (대시 위치 포함) |
