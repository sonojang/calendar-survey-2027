## 4️⃣ 데이터베이스 설계

**파일**: `supabase_schema.sql`

5개 테이블 + 인덱스 + RLS 정책:

| 테이블 | 용도 | 주요 컬럼 |
|--------|------|-----------|
| `domestic_quantities` | 국내 팀장 응답 | division, team, jangkum_b_qty, heunga_b_qty |
| `overseas_quantities` | 해외 시트2 | country, region, calendar_type(A/B), jangkum_qty, heunga_qty, yjc_*, shipping_method, address, port_code, pic_* |
| `detail_ports` | 해외 시트3 | country, region, port_code, quantity, address, company_name |
| `overseas_holidays` | 해외 시트4 | country, holiday_date, weekday, holiday_name_en |
| `network_changes` | 네트워크 변경 | country, branch_name, field, old_value, new_value, full_note |

### RLS (Row Level Security) 정책
- 익명(anon) 사용자: **INSERT만 허용** (응답 제출)
- 인증된 사용자(authenticated, 즉 관리자): **전체 권한**

→ publishable key를 사이트에 노출해도 무단 조회/삭제 불가

## 의사결정 사항 정리

| 항목 | 결정 | 대안 |
|------|------|------|
| DB | Supabase | Google Sheets(Apps Script), EmailJS 후보였음 |
| 정적 vs SSR | 정적 HTML | Node.js Express도 가능했으나 호스팅 부담 |
| 관리자 보호 | Supabase Auth 이메일/비밀번호 | URL 비밀번호도 후보 |

| 응답 저장 | 모든 응답 누적, 최신 응답이 엑셀에 반영 | 응답 1회 제한 / Upsert도 가능 |
| 엑셀 양식 | 원본 4시트 양식 + 네트워크 변경 5번째 시트 | CSV만 제공도 가능 |
