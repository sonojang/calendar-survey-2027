# 세부포트/공휴일/네트워크 - 임시저장 + 이전 제출 불러오기 (2026-07-06)

기존에 국내신청·해외수량·발송등록에만 있던 **임시저장 + 이전 제출 불러오기·수정** 기능을
이제 해외 페이지의 나머지 세 탭에도 적용했습니다.

- ② 세부포트
- ③ 공휴일
- ④ 네트워크 변경사항

---

## 🚨 배포 전 반드시 실행: Supabase DB 정책 추가

세 탭은 반복 행 방식(여러 건을 한번에 편집)이라 **일부 행 삭제**가 필요합니다.
이를 위해 anon DELETE 정책이 있어야 합니다.

### 실행 방법

1. Supabase Dashboard → SQL Editor
2. **Ctrl+A로 기존 내용 비우고**, 아래 파일 내용을 통째로 붙여넣기
   - `supabase_v8_anon_delete.sql`
3. **Run**

한 번만 실행하면 됩니다. `if exists` 로 되어있어 재실행해도 안전합니다.

> ⚠️ 만약 SQL 에러가 반복되면 (Supabase가 이전 SQL을 자동 복원하는 이슈), 
> Table Editor GUI에서 각 테이블(detail_ports, overseas_holidays, network_changes) 마다
> **RLS Policies → New policy → Allowed operation: DELETE, Target roles: anon, USING: true**
> 로 수동 추가하세요.

## 사용자 흐름

### 💾 임시저장

각 탭 하단의 **💾 임시저장** 버튼 → 브라우저에 로컬 저장 (같은 브라우저 기준 유지)

페이지 다시 열면 상단에 **"임시저장된 자료가 있습니다"** 배너가 뜹니다.
→ [불러오기] 클릭하면 폼에 값이 채워집니다.

### 📝 이전 신청 조회·수정

각 탭 상단의 **📝 내 이전 신청 조회·수정** 버튼:

1. 이메일 입력 (신청 시 사용한 이메일)
2. 조회 → **국가별(세부포트/공휴일) 또는 회사별(네트워크)** 그룹으로 표시
3. 원하는 그룹의 [불러오기] 클릭 → 그 그룹의 모든 행이 폼에 채워짐
4. **행 편집·추가·삭제** 후 [전체 제출] 클릭
5. 서버에서 기존 항목 **일괄 갱신** (수정된 것은 UPDATE, 지운 것은 DELETE, 새로 추가한 것은 INSERT)
