## ✅ Step 2 — eta 컬럼 추가 + RLS 정책 설정 (2분)

### 2-1. eta 컬럼 추가
같은 Tables 페이지에서:
1. `shipping_status` 테이블 클릭
2. **+ New column** 클릭
3. **Name**: `eta`, **Type**: `text`, Nullable ✅ → **Save**

### 2-2. RLS 정책 설정
좌측 메뉴 **Authentication** → **Policies** → `shipping_status` 테이블 찾기 → **New Policy** 클릭

다음 3개 정책을 만듭니다 (각각 별도로):

**정책 1: 익명 INSERT 허용** (총무팀 발송 등록용)
- Policy name: `anon insert shipping`
- Allowed operation: **INSERT** ✅
- Target roles: **anon**
- WITH CHECK expression: `true`
- Save

**정책 2: 익명 SELECT 허용** (신청자 조회용)
- Policy name: `anon select shipping`
- Allowed operation: **SELECT** ✅
- Target roles: **anon**
- USING expression: `true`
- Save

**정책 3: 관리자 전체 권한**
- Policy name: `auth all shipping`
- Allowed operation: **ALL** ✅ (또는 SELECT/INSERT/UPDATE/DELETE 모두)
- Target roles: **authenticated**
- USING expression: `true`
- WITH CHECK expression: `true`
- Save
